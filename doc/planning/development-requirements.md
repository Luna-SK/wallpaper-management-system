# 开发需求修正版

## 用户管理模块

- 用户注册、登录、个人信息修改、用户自助改密和管理员重置密码；邮件找回属于后续增强能力。
- 基于角色分配权限，控制图片、分类、标签、用户、角色、日志、设置等资源操作。

## 图片上传与管理模块

- 支持单张和批量上传墙布图片，上传前校验文件类型、大小和权限。
- 图片分类表示主归属，一张图片最多选择一个分类。
- 标签通过标签组组织，标签组表示描述维度，例如瑕疵、风格、颜色、材质；图片可选择多个任意启用标签。
- 一期保留 `纺织瑕疵` 分类，原 35 个瑕疵标签归入 `瑕疵` 标签组。
- 图片编辑包含元数据编辑、分类调整、标签调整；在线裁剪、旋转、缩放、水印属于后续增强能力。

## 图片存储与备份模块

- 图片对象存储在 RustFS 或兼容 S3 的对象存储中。
- 上传后保留原图，并生成缩略图、标准预览图和高清预览图。
- 数据库通过 Docker MySQL 备份；对象存储备份策略在部署文档中维护。

## 图片检索与浏览模块

- 支持关键词搜索、分类筛选、标签筛选和图片预览。
- 标签筛选不依赖分类，可按标签组展示和选择。
- 图片详情展示大图、分类、标签、尺寸、文件名、访问量、下载量等信息。

## 权限控制与安全模块

- 按角色限制图片访问和管理操作。
- 操作日志记录登录、上传、编辑、下载、删除、分类标签维护和系统设置变更。
- 敏感配置不提交到版本库；账号密码等敏感数据使用哈希或加密存储。
- 图片版权保护以鉴权下载、访问审计和预览图控制为基础，水印属于后续增强能力。

## 系统管理与维护模块

- 统计图片数量、上传量、访问量、下载量和存储占用。
- 系统设置维护上传限制、预览质量、软删除清理策略和审计归档策略。
- 日志管理支持近期查询、归档和保留期清理。
- 数据库结构以 Liquibase 为唯一真相源，不启用运行时自动建表或改表。

## 用户交互与反馈模块

- 评论、收藏、点赞和用户反馈属于后续增强模块，不进入当前一期核心交付。
- 后续实现时应复用用户、图片、权限和审计基础能力。
