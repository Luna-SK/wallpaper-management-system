# 图片导入计划

## Input

当前正式导入工具位于 `image-uploader/`。工具读取 `DATA_DIR` 下的图片目录。当前数据集使用 `0` 到 `20` 的文件夹编号组织纺织瑕疵图片。新系统把这些编号映射为已有 `纺织瑕疵` 分类和已有 `瑕疵` 标签组下的标签名，不把编号写死在图片表中。

## Flow

1. 导入器扫描 `0` 到 `20` 编号目录和图片扩展名。
2. 导入器根据文件夹编号得到一个或多个标签名，并生成友好图片标题。
3. 导入器调用新项目登录、分类/标签查询和上传会话 API，不直连数据库。
4. 上传 item 时直接提交标题；确认上传会话后由后端完成文件校验、上传去重、RustFS 写入、元数据入库和标签绑定。
5. 导入器写入本地 checkpoint 和 CSV 报告，输出成功、重复、失败和跳过数量。

## Acceptance

- 默认 dry run 不上传文件。
- 重复文件是否入库由后端“上传去重”开关决定，导入器不做本地图片判重。
- 任一导入失败不影响后续文件。
- 导入结果包含可追踪的源文件路径、期望标题、后端图片 ID 和失败原因。
